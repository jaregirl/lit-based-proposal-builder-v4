# v4.6.0 Pilot Learner and User Experience Evaluation

## Scope and Evidence

This formative evaluation used the current v4.6.0 build, automated browser walkthroughs, and fresh screenshots at 1440 x 900, 768 x 1024, and 390 x 844. It assessed the planned SHS, bachelor's, master's, doctoral, individual, group, and ERB-relevant paths. No real student or adviser feedback has been collected for v4.6.0 yet; the anonymous per-step feedback report is ready for the pilot.

## Overall Judgment

The app now functions as a guided research-learning workflow rather than a collection of disconnected forms. Its strongest learning sequence is:

`identify -> synthesize -> distinguish -> justify -> align -> reflect -> seek adviser review`

This progresses from foundational recognition to analysis, evaluation, and defensible design choices. It also supports integration and learning-how-to-learn through carried-forward evidence, revision prompts, readiness reflection, and adviser review.

## Scenario Walkthroughs

## Learning Support

- **Research-learning support:** Strong. A1-A4 requires synthesis rather than article-by-article summary, and the evidence chain links a question to its claim, evidence, source, instrument, and analysis.
- **Scaffold clarity:** Strong. Help appears near the task, examples are collapsible, and the A4 and methodology decisions are progressively disclosed.
- **Design-choice understanding:** Stronger than v4.5.x. The app recommends broad evidence approaches from transparent local rules, shows fit and assumptions, and still requires student justification.
- **Transfer:** Strong. Outputs carry the learner's reasoning into progress PDFs, adviser review, instrumentation, analysis, and proposal structure.
- **Reflection:** Strong. Initial/final/change reflections make revision visible without treating app activity as research quality.

## User Experience

- **Next-step visibility:** Good. Numbered stages and Back/Next actions remain visible at all tested widths.
- **Cognitive load:** Moderate but appropriate to the task. Long ethics and group forms use collapsed sections; the mobile navigation remains dense but scannable.
- **Error recovery:** Strong. Autosave, five checkpoints, recovery-before-restore, portable backups, and confirmations protect substantial work.
- **Adviser efficiency:** Strong. Review blocks, clean final output, and a separate readiness report distinguish student work from automated guidance.
- **Responsive layout:** Passed the tested desktop, portrait, and mobile widths without covered inputs, overlapping action bars, or clipped controls.

## Findings by Priority

### Critical

No critical barrier was found in the automated and visual walkthroughs.

### Important for the Pilot

1. The ERB DOCX files passed structural token and ZIP checks, but visual page-by-page comparison could not be completed on this machine because LibreOffice is unavailable. Open all four drafts in Microsoft Word before public release.
2. The full end-to-end workflow is necessarily long. Pilot instructions should encourage students to work by stage, create checkpoints, and consult their adviser rather than complete it in one sitting.
3. Methodology recommendations are developmental prompts, not validation. The current adviser language is necessary and should remain.

### Future Evidence-Based Revisions

1. Review anonymous per-step ratings after the first pilot and identify stages with low clarity/helpfulness or high difficulty.
2. Conduct keyboard-only and screen-reader sessions with real users; screenshots cannot establish accessibility compliance.
3. Replace blocking browser confirmation dialogs with accessible in-page confirmations if users report disruption.
4. Consider further collapsing the mobile stage list only if pilot users report navigation fatigue; the present numbered list preserves process visibility.

## Release Recommendation

The build is suitable for a controlled v4.6.0 pilot after the four ERB Word drafts receive a final visual check in Microsoft Word. Keep v3.3 available for students already using it. Treat learner and adviser feedback from this pilot as the evidence base for the next release rather than adding more features immediately.
